================================================================================
                        SPI TRANSACTION FORMAT (SD CARD)
================================================================================

SPI Frame Structure (8-bit):
┌──┬──┬──┬──┬──┬──┬──┬──┐
│7 │6 │5 │4 │3 │2 │1 │0 │ ← Bit Position (MSB first)
├──┼──┼──┼──┼──┼──┼──┼──┤
│  │  │  │  │  │  │  │  │ ← Data (MSB sent first)
└──┴──┴──┴──┴──┴──┴──┴──┘

SD Card Command Format (6 bytes):
┌────┬────────────────────────────────────────────────┬──────────┐
│ CMD│ ARG (4 bytes - 32 bits)                        │ CRC (1b) │
├────┬──┬──────────────────────────────────────────────┼──────────┤
│Byte│0 │1          │2         │3         │4          │5         │
├────┼──┼──────────────────────────────────────────────┼──────────┤
│Bits│47│46-32      │31-24     │23-16     │15-8       │7-1   │0 │
├────┼──┼──────────────────────────────────────────────┼──────────┤
│Data│01│Command    │Argument (MSB first)              │CRC7 │End│
│    │  │(6 bits)   │                                 │     │Bit│
└────┴──┴──────────────────────────────────────────────┴──────────┘

Example: CMD0 (GO_IDLE_STATE)
├─ Byte 0: 0x40 (01 000000) = CMD0
├─ Bytes 1-4: 0x00 0x00 0x00 0x00 (no arguments)
├─ Byte 5: 0x95 (10010101) = CRC + end bit
└─ Total: 6 bytes transmitted

SD Card Response Format (R1):
┌───┬───┬───┬───┬───┬───┬───┬───┐
│7  │6  │5  │4  │3  │2  │1  │0 │
├───┼───┼───┼───┼───┼───┼───┼───┤
│Bit│ Meaning                    │
├───┼──────────────────────────────┤
│7  │ Start Bit (always 0)        │
│6  │ Parameter Error             │
│5  │ Address Error               │
│4  │ CRC Error                   │
│3  │ Illegal Command             │
│2  │ Erase Sequence Error        │
│1  │ SPI Mode Error              │
│0  │ Idle State (1=idle, 0=busy) │
└───┴──────────────────────────────┘

Example: R1 Response 0x01
├─ Bit 0: 1 (Idle)
├─ Bit 1-6: 0 (No errors)
├─ Bit 7: 0 (Start bit)
└─ Interpretation: SD card is ready
